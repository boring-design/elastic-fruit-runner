<Aside type="tip" title="One Mac, three platforms">
A single MacBook Pro or Mac mini with Apple Silicon can simultaneously serve as:
- **macOS arm64** runners via ephemeral Tart VMs
- **Linux arm64** runners via Docker containers
- **Linux amd64** runners via Docker + Rosetta 2

No more GitHub billing anxiety. No more idle hardware. Just `brew install` and go.
</Aside>

## Features

<CardGrid stagger>
	<Card title="macOS Runners (Tart)" icon="apple">
		Ephemeral macOS VMs on Apple Silicon via [Tart](https://tart.run). One VM per job, auto-destroyed after completion. Build and test iOS/macOS apps on real Apple hardware.
	</Card>
	<Card title="Linux Runners (Docker)" icon="linux">
		Linux arm64 and amd64 runners via Docker-in-Docker. amd64 runs through Rosetta 2 — one Mac covers all your CI architectures.
	</Card>
	<Card title="Scale to Zero" icon="down-caret">
		No idle VMs, no idle containers. Runners are provisioned on demand and destroyed after each job. When there are no jobs, resource consumption is near zero.
	</Card>
	<Card title="No Kubernetes Required" icon="seti:config">
		No minikube, no k3s, no cluster management. Just `brew install`, write a config file, and start the service. Goodbye, YAML manifests.
	</Card>
	<Card title="Official Scale Set API" icon="github">
		Powered by the official [GitHub Runner Scale Set API](https://github.com/actions/scaleset). One daemon manages all your orgs and repos — no per-repo runner registration, no token juggling.
	</Card>
</CardGrid>
